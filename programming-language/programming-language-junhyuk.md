### 1. CSMA/CA

1. **CSMA/CD (Collision Detection)**: 이 프로토콜은 주로 유선 이더넷 네트워크에서 사용됩니다. CSMA/CD는 데이터 전송 중 충돌이 감지되면 전송을 중단하고 임의의 시간 동안 기다린 후 다시 전송을 시도합니다. 전송 장치는 채널이 사용 중인지 감지하고, 사용 중이 아니면 데이터를 전송합니다. 데이터 전송 중에 충돌이 발생하면, 전송 장치는 전송을 중단하고 재전송을 위해 임의의 대기 시간을 갖습니다.
2. **CSMA/CA (Collision Avoidance)**: 이 프로토콜은 주로 무선 네트워크, 특히 Wi-Fi에서 사용됩니다. CSMA/CA는 충돌을 감지하는 대신 충돌을 피하기 위한 메커니즘을 사용합니다. 전송 장치는 채널이 사용 중인지 확인한 후, 데이터를 전송하기 전에 잠시 대기하는 "대기 시간"을 할당합니다. 이는 주변 장치에게도 데이터 전송 의도를 알리고 충돌 가능성을 최소화합니다. 또한, 전송 장치는 "ACK" (Acknowledgement) 패킷을 받을 때까지 데이터 패킷을 재전송할 수 있습니다.

- External Coupling은 모듈이 외부 데이터나 함수를 사용하는 경우에 발생하는 결합도. 모듈 간의 의존성은 없지만 외부 데이터나 함수의 변경에 민감하기에 Common coupling 보다는 약한 결합도.
